**Failover** — это производительный и отказоустойчивый менеджер VPN-подписок, написанный на Rust. Приложение полностью автоматизирует управление прокси-серверами: самостоятельно подбирает оптимальные конфигурации на основе задержки и нагрузки, запоминает наиболее стабильные профили и параллельно распределяет трафик разных сайтов через разные прокси.

> **🤖 Разработка:** Данная программа и её архитектура полностью спроектированы и сгенерированы с помощью искусственного интеллекта (ИИ).

> **⚠️ Статус проекта:** В настоящее время доступен полнофункциональный консольный интерфейс (**CLI**). Разработка кроссплатформенного графического интерфейса (**GUI**) находится в планах.

---

## Основные возможности

* **Умный выбор профилей (Smart Selection):** Автоматический подбор конфигурации по принципу наименьшей задержки (ping) и оптимальной нагрузки.
* **История стабильности (Profile Scoring):** Failover анализирует поведение серверов со временем, запоминает «лучшие» профили и отдает им приоритет при последующих подключениях.
* **Параллельная маршрутизация (Multi-Profile Routing):** Возможность одновременно направлять трафик разных доменов и под-URL через независимые прокси-профили. Например:
* Трафик `YouTube` направляется через быстрый *Профиль А* (Германия).
* Запросы к `GitHub` идут через стабильный *Профиль Б* (Швеция).
* Специфические под-домены параллельно распределяются по другим узлам.


* **Автоматический Failover:** Бесшовное переключение на альтернативный рабочий профиль из пула, если текущий сервер перестал отвечать или деградировал по скорости.
* **Управление подписками:** Автоматическое обновление, парсинг, валидация и фильтрация входящих пулов прокси. *(Текущая интеграция с xray-knife является временной и в будущем будет заменена на собственное независимое решение).*

## Архитектура проекта

Проект построен по модульному принципу (Rust Workspace) для максимальной изоляции логики:

* `vpn-core` — ядро приложения, координация состояний и основная бизнес-логика.
* `vpn-subs` — модуль парсинга, валидации и обновления подписок.
* `vpn-testing` — движок тестирования серверов и анализа доступности узлов.
* `vpn-proxy` — управление прокси-процессами и маршрутизацией трафика.
* `vpn-l10n` — легковесный модуль встроенной локализации.
* `vpn-knife` — временный модуль низкоуровневого взаимодействия с движком *(готовится к удалению)*.

## Конфигурация (.json)

Вся логика работы приложения настраивается через JSON-файлы, расположенные в директории конфигурации:

* `config.json` — глобальные параметры приложения, тайм-ауты и настройки тестирования.
* `subscriptions.json` — список ваших VPN-подписок и источников.
* `parallel_rules.json` — правила параллельной маршрутизации трафика (маски сайтов, домены и под-URL с привязкой к профилям или регионам).
* `commands.json` — конфигурация кастомных команд и сценариев автоматизации.

## Использование (CLI)

```bash
# Обновление всех подписок из конфигурационного файла
vpn-manager subs update

# Запуск тестирования профилей и выбор оптимального узла
vpn-manager test --optimize

# Запуск прокси с применением правил параллельной маршрутизации
vpn-manager start --rules parallel_rules.json

```
